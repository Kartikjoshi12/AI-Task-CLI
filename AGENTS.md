# AI Coding Agents Guidelines

## Source of Truth
Always refer to these files for the project's intent, design, and progress:
- `PROJECT.md`: Vision, goals, and project scope.
- `SPEC.md`: Technical specifications and requirements.
- `CLI.md`: CLI interface and command definitions.
- `STORAGE.md`: Storage architecture and data models.
- `ROADMAP.md`: Implementation order and milestones.

## Operating Principles
- **Specification Integrity**: Never change the specification without explicit approval.
- **Incremental Development**: Build one feature at a time.
- **Granular Commits**: Keep commits small and focused.
- **Simplicity**: Prefer simple solutions; do not over-engineer.
- **Dependency Control**: Always ask before introducing new dependencies.
- **Architecture**: Keep the project provider-based and storage-independent to ensure flexibility.
